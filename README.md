> A curated list of awesome tools, frameworks, practices, and resources for Infrastructure as Code (IaC).

Infrastructure as Code (IaC) is the practice of managing and provisioning computing infrastructure through machine-readable configuration files, rather than manual processes.

## Getting Started

- [Infrastructure as Code – ThoughtWorks](https://www.thoughtworks.com/radar/techniques/infrastructure-as-code) – Overview and strategic importance.
- [Terraform Getting Started](https://developer.hashicorp.com/terraform/docs) – Official documentation from HashiCorp.
- [Pulumi Introduction](https://www.pulumi.com/docs/get-started/) – Infrastructure as code in general-purpose languages.

## IaC Tools

- [Terraform](https://www.terraform.io/) – Tool from HashiCorp for provisioning infrastructure.
- [Pulumi](https://www.pulumi.com/) – Write infrastructure in Python, TypeScript, Go, .NET.
- [AWS CloudFormation](https://aws.amazon.com/cloudformation/) – AWS-native IaC solution using YAML/JSON templates.
- [Azure Resource Manager (ARM)](https://learn.microsoft.com/en-us/azure/azure-resource-manager/) – Native IaC for Microsoft Azure.
- [Google Cloud Deployment Manager](https://cloud.google.com/deployment-manager) – Native GCP infrastructure automation.
- [Crossplane](https://crossplane.io/) – Kubernetes-native IaC and control plane.
- [CDK (Cloud Development Kit)](https://cdk.dev/) – Define cloud infrastructure using programming languages.

## Configuration Management

- [Ansible](https://www.ansible.com/) – Agentless configuration management and automation platform.
- [Chef](https://www.chef.io/) – Infrastructure automation framework using Ruby.
- [Puppet](https://puppet.com/) – Configuration management and automation for system administration.
- [SaltStack](https://saltproject.io/) – Event-driven automation and configuration.

## Policy as Code

- [Open Policy Agent (OPA)](https://www.openpolicyagent.org/) – Unified tool for policy enforcement across the stack.
- [HashiCorp Sentinel](https://www.hashicorp.com/sentinel) – Policy as code framework integrated into HashiCorp tools.
- [Conftest](https://www.conftest.dev/) – Validate configuration files using OPA/Rego policies.
- [Terralinter](https://github.com/accurics/terrascan) – Security and compliance scanner for Terraform.

## Testing Tools

- [Terratest](https://github.com/gruntwork-io/terratest) – Go-based integration testing framework for IaC.
- [Kitchen-Terraform](https://github.com/newcontext-oss/kitchen-terraform) – Integration testing with Test Kitchen.
- [Checkov](https://github.com/bridgecrewio/checkov) – Security-focused static analysis for Terraform, CloudFormation, etc.
- [tflint](https://github.com/terraform-linters/tflint) – Linter for Terraform code.
- [cfn-lint](https://github.com/aws-cloudformation/cfn-lint) – Linter for AWS CloudFormation templates.

## CI/CD Integration

- [Atlantis](https://www.runatlantis.io/) – Terraform pull request automation tool.
- [Spacelift](https://spacelift.io/) – Collaborative IaC management platform.
- [GitHub Actions for Terraform](https://github.com/hashicorp/setup-terraform) – Official GitHub action to integrate Terraform.
- [CircleCI + Terraform](https://circleci.com/blog/automate-your-infrastructure-with-terraform-and-circleci/) – Guide for integrating Terraform in CircleCI.
